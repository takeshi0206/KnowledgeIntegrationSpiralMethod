# Knowledge Integration Spiral Method (KISM)

**ナレッジスパイラル：持続可能なシステム運用を支える7段階プロジェクト管理手法の提案**

*Projects end. Systems don't.*
*Tackling that asymmetry with structure, not slogans.*

---

## What is KISM?

KISM (Knowledge Integration Spiral Method) は、「プロジェクトは終わるが、システムは動き続ける」という非対称性に構造的に対処するプロジェクト管理手法です。

「コミュニケーションを改善しましょう」という曖昧な処方箋ではなく、**プロセス設計と構造的仕組み**によって、将来の保守担当者が安全にシステムを変更できる状態を作ります。

### BRIEFED — 7段階プロセス

| 段階 | 名称 | 目的 |
|------|------|------|
| **B** | Broadcast | ルールと期待値の周知、決定権限の明確化 |
| **R** | Realize | ルールの運用、実態の記録 |
| **I** | Identify | 問題の特定と定義 |
| **E** | Elaborate | 対策案の作成、法的根拠の確認 |
| **F** | Feedback | レビュー、保守性の検証 |
| **E** | Establish | ドキュメント化、設計意図の記録 |
| **D** | Disseminate | 合意形成、次サイクル・保守フェーズへの継承 |

### 主な特徴

- **システムライフサイクル視点** — プロジェクト完結ではなく、システムの長期運用を視野に入れた設計
- **心理的障壁への構造的対策** — 学生症候群、決定回避、現在バイアス等への仕組みによる対処
- **構造による解決** — 個人のスキルや関係性に依存しない、再現性のある仕組み
- **AI協働の枠組み** — 各段階でAIが担う領域と人間が担う領域を構造的に定義
- **KISM規約** — そのままプロジェクトの運用ルールとして採用できる実践的な規約

### 対象領域

本手法は「正しく動き続けなければならないシステム」を対象とします：

- 公共の事務処理システム（住民税計算、給与支払い等）
- 業務基幹システム
- 法令に基づく処理システム

創造性や革新性よりも、**正確性・確実性・継続性**が求められる領域に特化しています。
